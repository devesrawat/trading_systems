= NSE Algorithmic Trading System — Usage Guide
:toc: left
:toc-title: Contents
:toclevels: 3
:icons: font
:source-highlighter: highlight.js

A walk-forward XGBoost/LightGBM/PatchTST ensemble signal model backed by FinBERT sentiment, connected to Zerodha Kite Connect, with the Hermes dual-engine (Momentum Sentinel + Wealth Architect) and hard risk guardrails enforced by a circuit breaker.

See link:../README.md[README.md] for the full usage guide and project overview.

---

== Prerequisites

=== Infrastructure

[cols="1,2,2", options="header"]
|===
| Service | Purpose | Default address

| TimescaleDB (PostgreSQL) | OHLCV data, trades, signals, audit log | `localhost:5432`
| Redis | Tick cache, circuit-breaker state, access token, fundamentals cache | `localhost:6379`
| MLflow | Experiment tracking, model registry | `localhost:5001`
|===

Start all three with Docker Compose:

[source,bash]
----
docker compose up -d timescaledb redis mlflow
----

Initialise the schema (hypertables + audit tables) once after first launch:

[source,bash]
----
python -m data.store --init-schema
----

=== Python

Requires Python ≥ 3.13. Install dependencies with `uv` (recommended):

[source,bash]
----
uv sync
# For Upstox support:
uv sync --extra upstox
----

=== Environment variables

Copy the example file and fill in the required fields:

[source,bash]
----
cp .env.example .env
----

[source,dotenv]
----
KITE_API_KEY=your_api_key_here
KITE_API_SECRET=your_api_secret_here
TIMESCALE_URL=postgresql://trader:password@localhost:5432/nse_trading
REDIS_URL=redis://localhost:6379/0
MLFLOW_TRACKING_URI=http://localhost:5001
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id
----

See link:env-variables.adoc[env-variables.adoc] for the full variable reference.

== First-time setup

=== 1. Kite login

[source,bash]
----
python -m data.ingest --login
----

Opens a browser to Zerodha OAuth. After authorising, the session token is cached in Redis. Subsequent logins are handled automatically each morning at 08:45 IST.

=== 2. Initialise instrument universe

[source,bash]
----
python -m data.universe --refresh
----

Downloads the NSE instrument list from Kite Connect. Re-run at weekends when NSE updates instruments.

=== 3. Backfill historical OHLCV

[source,bash]
----
python -m data.ingest --backfill --days 1095
----

Pulls daily bars for the top 50 liquid instruments (≈3 years). Takes 10–20 minutes on first run.

=== 4. Populate Wealth Architect fundamentals cache

The conservative leg of Hermes requires per-symbol fundamentals in Redis:

[source,python]
----
import json, redis
r = redis.from_url("redis://localhost:6379/0")
r.set("FUND:INFY", json.dumps({
    "pe": 24.5, "roe": 29.0,
    "sector": "IT", "sector_avg_pe": 28.0
}))
----

Run this via the `fundamentals/` fetchers which populate `FUND:{symbol}` keys automatically.

== Paper trading (recommended starting point)

Paper mode is the default. Orders are logged to `paper_trades` and never sent to Kite.

=== Start the system

[source,bash]
----
python -m orchestrator.main
----

The scheduler starts and immediately logs registered jobs. Use `Ctrl+C` to stop.

=== Market mode

[source,bash]
----
python -m orchestrator.main --market equity   # NSE weekdays IST
python -m orchestrator.main --market crypto   # Binance 24/7 UTC
python -m orchestrator.main --market both     # both simultaneously
----

=== Verify it is working

[source,bash]
----
# Check recent paper trades
python -m execution.logger --summary --date today

# Check circuit breaker state
python -m risk.breakers --status

# Check active signal model
python -m signals.model --status
----

=== Accumulate paper trades

After 200+ paper trades, review the metrics:

[source,bash]
----
python -m execution.logger --summary --days 30
----

A win rate above 50 % and a mean P&L per trade above transaction costs is the minimum bar before considering live trading.

== Live trading

[WARNING]
====
Live mode places real orders with real money. Do not enable it until:

. 200+ paper trades completed with satisfactory metrics
. `DAILY_DD_LIMIT` and `MAX_POSITION_PCT` reviewed in `.env`
. Circuit breaker behaviour tested in paper mode (verify it halts correctly)
====

In `.env`, set:

[source,dotenv]
----
PAPER_TRADE_MODE=false
----

Then restart the system. The startup log emits `live_trading_mode_active` to confirm.

== The Hermes Dual-Engine

Hermes is the strategy layer with two legs operating concurrently.

=== Leg 1: Momentum Sentinel (Aggressive)

Scans for high-momentum breakout candidates. Runs as a `BaseStrategy` subclass (pure CPU — no I/O, no Redis) inside the `ScannerEngine` ProcessPoolExecutor.

*Criteria:*

* Close > SMA(50)
* Volume > 2× 20-day average volume
* RSI(14) > 60 (Wilder-smoothed)

*Sort order:* highest `volume_ratio` first.

=== Leg 2: Wealth Architect (Conservative)

Weekly scan for blue-chip compounding candidates. Runs standalone (reads Redis — not allowed in worker processes).

*Criteria:*

* PE ratio < sector average PE (fallback cap: 25)
* ROE > 15%

*When it runs:* Saturday 09:00 IST. Top-3 candidates sent to Telegram.

=== AlphaEngine (Composite Multiplier)

Applies a scalar multiplier (0.5× – 1.5×) to the base model probability:

[cols="1,1,1", options="header"]
|===
| Regime | BUY delta | SELL delta

| `trending_bull` | +0.10 | −0.10
| `trending_bear` | −0.30 | +0.10
| `choppy`        | −0.20 | −0.10
| `high_vol`      | −0.10 |  0.00
| `normal`        |  0.00 |  0.00
|===

Final confidence = model probability × AlphaEngine multiplier.

== Model training

=== Train from scratch

[source,bash]
----
python -m signals.train --symbol RELIANCE --start 2021-01-01 --end 2024-01-01
----

Uses walk-forward cross-validation (24-month train, 3-month test, 5-day purge gap). Results and the best model are logged to MLflow.

=== Register the model for production

After reviewing the run in the MLflow UI (`http://localhost:5001`), promote the model:

[source,bash]
----
python -m signals.model --register --run-id <mlflow_run_id> --alias production
----

The orchestrator loads the `production` alias at each pre-market setup.

=== Retrain trigger

The Sunday retrain check compares the live 30-day win rate against the backtest baseline. If drift exceeds threshold, a Telegram alert is sent. Retraining is manual — run the train command and re-register.

== Circuit breaker

The circuit breaker halts all new orders when any of three conditions is met:

[cols="1,3", options="header"]
|===
| Condition | Default limit

| Daily drawdown | 3 % of daily starting capital (`DAILY_DD_LIMIT`)
| Weekly drawdown | 7 % of weekly starting capital (`WEEKLY_DD_LIMIT`)
| Consecutive losses | 5 losses in a row (`max_consecutive_losses` in `risk/breakers.py`)
|===

=== Check status

[source,bash]
----
python -m risk.breakers --status
----

=== Manual reset

Drawdown halts **never** auto-reset. After investigating the cause:

[source,bash]
----
python -m risk.breakers --reset
----

This updates both daily and weekly baselines to current capital and clears the halt. An audit record is written to `circuit_events`. The Monday weekly reset does *not* clear a halt — `--reset` must be called explicitly.

[source,bash]
----
